# AGENTS.md

The terse index every agent reads. Durable narrative lives in
[`docs/conventions/workflow.md`](docs/conventions/workflow.md); law lives in
[`CONSTITUTION.md`](CONSTITUTION.md) (copy it from `CONSTITUTION.template.md`); business
context lives in `docs/PROJECT.md` (generated by `/project-explore`).

> **Template.** Fill in the `<…>` and prune anything that doesn't fit your project.

## The two documents

- **`CONSTITUTION.md`** — engineering law (stack lock, hard rules, DoD). Enforced by agents.
- **`docs/PROJECT.md`** — business context (idea, goals, roles, scenarios). Keeps the goal in view.

Never conflate them.

## Feature workflow

Four commands are the whole external API. Detail: `docs/conventions/workflow.md`.

| Command | When | Does |
|---|---|---|
| `/project-explore` | once per project + to refresh | survey → `docs/PROJECT.md` + `docs/architecture-map.md` |
| `/feature-start` | begin a feature | size/route → recon → spec-analyst → devils-advocate → Plan Mode → `plan.md` |
| `/feature-finish` | implementation done | code-reviewer → test gate → DoD vs diff → docs → drafts a PR, then STOPS |
| `/fix` | a bug | reproduce → failing test → smallest fix → same gate → record |

**Push / PR are human-only.** The agent prepares; a human presses (`docs/conventions/git.md`).

## Agent roster

Read-heavy and adversarial work runs in subagents so the main window stays on decisions.
Adjust models to your budget.

| Agent | Role | Suggested model |
|---|---|---|
| `codebase-explorer` | read-only recon, reports `path:line` | cheap/fast |
| `spec-analyst` | reconcile spec vs constitution + code, surface clarifying questions | mid |
| `architect` | design notes, ADRs, schema/architecture deltas | strong |
| `devils-advocate` | last gate before Plan Mode — clean context, cite-or-drop | strong |
| `implementer` | write code against an approved plan | mid/strong |
| `code-reviewer` | pre-PR review across design/readability/security/perf/testability | strong |
| `test-runner` | run the quality gate (lint → typecheck → test → build) | cheap/fast |
| `e2e-planner` | explore a live app, write a Markdown test plan | mid |
| `e2e-runner` | execute a Markdown test plan live, no generated spec files | mid |

## Surface placement

Where new code lives — adapt to your layout:

- Apps → `apps/*` · Feature modules → `modules/*` · Shared packages → `packages/*`
- Before adding surface, check `CONSTITUTION.md` for the canon and ownership rules.

## Rules & guardrails

- Always-on reminders: [`.agents/rules/engineering.md`](.agents/rules/engineering.md).
- Path-scoped rules load automatically: `typescript`, `validation-boundaries`, `database`,
  `packages`, `modules`, `testing` (see `.agents/rules/`).
- Deterministic guards (not guidance): `.agents/hooks/{guard,format}.sh`, wired in
  `.claude/settings.json`.
- Process validator: `pnpm validate:workflow` (or `node scripts/validate-workflow.mjs`).
